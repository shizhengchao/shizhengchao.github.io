GIT使用笔记

发表于 2017-03-24 | 分类于 笔记 | 阅读次数:

初始化工程

  1. 在Git官网上点击New repository新建项目
  2. 在本地新建一个同名文件。
  3. 打开git bash切换到新建的文件目录
  4. 新建一个README文件并写入内容
    echo "# myprojectname">>README.md
  5. 生成“.gitignore”文件。
    touch .gitignore
    写入 :
    .settings/
    target/
    .classpath
    .project
    .class
  6. git初始化
    git init
  7. 将全部文件添加到git中
    git add . #添加当前目录下的所有文件
git add README.md #只添加README.md文件
  8. 提交，-m参数添加注释
    git commit -m "commit message"
  9. 创建远程仓库origin
    git remote add origin git@github.com:shizhengchao/hello.git
如果出现: fatal: remote origin already exists. 说明远程仓库已经存在,如果想重新生成，执行git remote rm origin后，再新增。如果不想重新生成,则不用管,直接跳过这一步就行。
  10. push代码更新远程仓库
    git push -u origin master
  11. 9和10可以合并为:
    git push git@github.com:shizhengchao/hello.git master

常用操作

  1. 删除远程仓库中的文件和文件夹
    git rm --cached ./xxx/xxx.log   #删除文件
git rm -r --cached ./xxx/log/   #删除文件夹
  2. clone工程
    git clone xxx.git   #完整的拉取git仓库里的工程
  3. pull
    git pull   #远程获取最新版本并merge（合并）到本地
