JAVA责任链模式

发表于 2017-03-25 | 分类于 设计模式

前言

责任链模式是由命令对象和一系列处理对象组成的设计模式。 每个处理对象都包含用于定义它可以处理的命令对象类型的逻辑; 其余的被传递给链中的下一个处理对象。 也可以将新的处理对象添加到该链的末尾。
简单的说就是:请求从一端进入并继续从一个对象到对象，直到找到合适的处理程序。

适用性

什么情况下使用责任链

  • 一个请求被多个对象处理时,请求对和处理对象是不可知的，需要处理程序自动确定。
  • 多个对象之一发出请求，而无需明确指定接收者。
  • 可以动态地指定 可处理请求的对象集合。

经典案例

  • java.util.logging.Logger#log()
  • Apache Commons Chain
  • javax.servlet.Filter#doFilter()

linux常用命令记录

发表于 2017-03-20 | 分类于 笔记

基本命令

  • df -h : 查看磁盘使用情况
  • du -h –max-depth=1 : 查看当前目录下的所有目录的使用情况 参数 –max-depth=1代表目录深度
  • find . -type f -size +800M -print : 找出当前目录下及子目录下的大于800M的文件
  • rm -rf xxx:-f强制删除，-r如果是目录，删除包含子目录的所有文件
  • mkdir folder:创建文件夹
  • cp abc.log /dir/:复制abc.log文件到/dir目录下
  • mv abc.log /dir:移动或者重命名文件
